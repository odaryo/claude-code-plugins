以下の観点でセキュリティレビューを実施してください:

## 1. 機密情報の漏洩チェック
- ハードコードされた API キー、トークン、パスワードがないか検索（パターン: `sk-`, `token=`, `password=`, `secret=`, `api_key=`, `Bearer ` など）
- `.env` ファイルやクレデンシャルファイルが .gitignore に含まれているか確認
- テストデータに本物の機密情報が含まれていないか確認

## 2. プロンプトインジェクション対策
- スキルの `$ARGUMENTS` がシェルコマンドに渡される箇所で適切にクォート/サニタイズされているか
- ユーザー入力が `eval`, `bash -c`, `sh -c` に直接渡されていないか
- スキルの指示文に外部コンテンツの無条件取得が含まれていないか
- エージェントの指示文が外部データを信頼しすぎていないか

## 3. 危険なコマンド実行
- フック内に破壊的コマンド (`rm -rf`, `git push --force`, `git reset --hard`) がないか
- フックが過剰な権限（sudo、ルートアクセス）を要求していないか
- ネットワークアクセスする場合、送信先が制限されているか

## 4. 権限とスコープ
- 副作用のあるスキルに `disable-model-invocation: true` が設定されているか
- フックの matcher が必要以上に広くないか（例: 全ツールイベントではなく特定ツールに限定）

## 5. レポート
- 重大度別（Critical / Warning / Info）に分類して報告
- 各問題に具体的な修正案を提示
